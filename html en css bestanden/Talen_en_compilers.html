Bij een computer worden talen op verschillende niveaus gebruikt, elk met een specifiek doel en gebruik. Hier zijn een aantal van die niveaus:
Machinecode: Dit is de laagste programmeertaal die direct begrepen wordt door de computerhardware. Het bestaat uit binaire instructies, en elke instructie vertegenwoordigt een specifieke bewerking op het niveau van de CPU (Central Processing Unit). Dit is dus niet leesbaar voor de gemmidelde gebruiker aangezien het uit eenen en en nullen bestaat.
Assembler: Assemblertalen zijn laag-niveau talen die een menselijk leesbare representatie bieden van machinecode. Ze vertegenwoordigen instructies in tekstvorm en worden gebruikt om programma's te schrijven die dicht bij de hardware draaien. Het is dus een taal die leesbaarder is dan de machinecode met het binaire stelsel maar is nog niet op het niveau van een programmeertaal.
Programmeertaal: Er zijn heel veel verschillende programmeertalen elke taal heeft zijn unieke toepassing de site die je nu bekijkt is bijvoorbeeld grotendeels in html geschreven. Html is namelijk erg handig voor het programmeeren van sites. Veel van deze talen lijken op elkaar maar ze zijn niet hetzelfde elke programmeertaal is namelijk uniek.

Een compiler is een softwareprogramma dat wordt gebruikt om broncode, geschreven in een programmeertaal, om te zetten in een uitvoerbaar bestand of een andere vorm van code die direct kan worden uitgevoerd door een computer. Het proces begint met de invoer van de broncode. Dit is leesbare code geschreven in een programmeertaal, zoals C, C++, Java, enz. Daarna begint de lexicale analyse, ook wel scannen genoemd. Hierbij wordt de broncode opgedeeld in afzonderlijke woorden of tokens. Een token kan een sleutelwoord, een variabelenaam, een operator of een ander element van de taal zijn. De broncode wordt dus opgedeeld in behapbare delen voor de computer. Na de lexicale analyse begint de syntax analyse. Hierbij analyseert de compilerde structuur van de code om te controleren of deze voldoet aan de grammaticaregels van de programmeertaal. Als de code niet correct is gestructureerd, zal de compiler fouten melden en dit vermelden aan de gebruiker. Nadat de syntaxis is geanalyseerd, wordt de semantische analyse uitgevoerd om te controleren of de code betekenisvolle instructies bevat. Dit is onder andere het controleren op ongeldige bewerkingen en nog veel meer dingen. Na de syntaxis gaat de compilers het resultaat fine tunen om zo een zo goed mogelijk resultaat te krijgen. Het laatste stadium van de compilatie is codegeneratie. Hierbij wordt de getransformeerde broncode omgezet in doelcode voor een specifieke doelcomputer of -platform. Dit kan bijvoorbeeld machinecode zijn of assemblercode afhankelijk van de doeltaal en het doelsysteem.
